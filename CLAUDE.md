# Project Instructions

## Version Bumping

Before pushing a PR branch, always bump the version in `package.json`:
- **Minor bump** (e.g. 0.2.0 → 0.3.0): new features, new config options, new UI, new game mechanics
- **Patch bump** (e.g. 0.2.1 → 0.2.2): bug fixes, refactors, docs, style changes, dependency updates

Include the version bump in the same commit as the final change, or as a separate commit before pushing.
